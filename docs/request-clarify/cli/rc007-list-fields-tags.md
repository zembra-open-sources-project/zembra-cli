# CLI List Fields Tags 需求澄清

日期：2026.04.27

## 需求目标

为 zembra-cli 增加 `list` 命令组，用于浏览当前数据库中的 tag 和 field 名称。

## 已确认命令

| 命令 | 说明 |
| --- | --- |
| `zembra-cli list tags -n 10 -a` | 列出 tags |
| `zembra-cli list fields -n 10 -a` | 列出 fields |

## 参数规则

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `-n` | `5` | 限制输出条目数量，必须为大于等于 `1` 的整数 |
| `-a` | `False` | 输出全部条目 |

当 `-a` 与 `-n` 同时出现时，以 `-a` 为准，忽略 `-n`。

## 输出规则

| 项目 | 结论 |
| --- | --- |
| 排序 | 按 `name ASC` 排序 |
| 输出字段 | 只输出 `name` |
| 显示形式 | 默认紧凑显示，一行内多个 item |
| 分隔符 | 两个空格 |
| 空数据 | 输出空内容，退出码为 `0` |
| 非法 `-n` | 输出错误，退出码非 `0` |

## 实现命名

代码层使用 `list_app` 避免和 Python 内置 `list` 混淆，CLI 展示名称仍为 `list`。

## 范围边界

### In Scope

| 范围 | 说明 |
| --- | --- |
| CLI 命令组 | 新增 `zembra-cli list` |
| 子命令 | 新增 `tags` 和 `fields` |
| 数据读取 | 读取当前配置数据库中的 tags 和 fields |
| 输出格式 | 单行紧凑输出名称 |
| 参数校验 | 校验 `-n` 非法值 |
| 测试 | 覆盖默认数量、全部输出、非法参数、空数据等场景 |

### Out of Scope

| 范围 | 说明 |
| --- | --- |
| 长列表格式 | 不实现类似 `ls -l` 的一行一个 item 输出 |
| JSON 输出 | 本需求只要求紧凑文本输出 |
| 搜索过滤 | 不增加按前缀或关键词过滤 |
| 反向排序 | 不增加倒序参数 |
| 输出 id 或时间 | 只输出 `name` |
