# Slash 命令即时帮助需求澄清

日期：2026.05.02

## 需求结论

在 `zembra-cli run` 的交互输入模式中，当用户输入内容以 `/` 开头时，立即显示现有斜杠命令候选列表和帮助说明。该能力只用于提示已有命令，不新增交互命令，不影响普通笔记录入、`@field` 解析和 `#tag` 解析。

## 已确认规则

| 问题 | 结论 |
| --- | --- |
| 触发条件 | 只在当前输入内容以 `/` 开头时触发 |
| 命令范围 | 只覆盖已有命令 |
| 展示形式 | 输入 `/` 后显示类似 Codex 的 prompt_toolkit 临时候选菜单 |
| 匹配规则 | 前缀匹配 |
| 普通笔记 | 不以 `/` 开头的输入不触发命令候选 |

## 命令候选范围

| 命令 | 说明 |
| --- | --- |
| `/help` | 显示交互帮助 |
| `/exit` | 退出交互会话 |

## 交互规则

| 输入状态 | 行为 |
| --- | --- |
| `/` | 显示全部已有斜杠命令和帮助说明 |
| `/h` | 只显示 `/help` 候选和帮助说明 |
| `/e` | 只显示 `/exit` 候选和帮助说明 |
| `/unknown` | 无匹配候选时不显示候选，提交后保持现有未知命令提示 |
| `note / text` | 作为普通笔记内容处理，不触发候选 |

## 本阶段范围

| 对象 | 操作 |
| --- | --- |
| `src/zembra_cli/interactive.py` | 增强交互输入读取逻辑，使 `/` 前缀输入能即时显示命令帮助 |
| `/help` 与 `/exit` | 作为唯一候选命令来源 |
| 输入候选 | 使用 prompt_toolkit 原生 completer 展示临时候选 |
| 测试 | 覆盖前缀匹配、候选展示和普通输入不触发 |

## 暂不包含

- 新增 `/clear`、`/list`、`/search`、`/delete` 等命令
- Textual 全屏 TUI
- 模糊匹配
- 在笔记正文中间输入 `/` 时触发候选
- Rich 表格打印式候选历史输出

## 验收标准

| 场景 | 预期 |
| --- | --- |
| 在空输入行输入 `/` | 在输入行下方展示包含 `/help` 和 `/exit` 的临时候选菜单 |
| 输入 `/h` | 只展示 `/help` 候选 |
| 输入 `/e` | 只展示 `/exit` 候选 |
| 输入普通笔记 | 不展示命令候选，保存逻辑保持不变 |
| 提交 `/help` | 继续沿用现有帮助命令行为 |
| 提交 `/exit` | 继续沿用现有退出行为 |
