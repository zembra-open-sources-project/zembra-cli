# Add 命令 role 参数需求澄清

日期：2026.04.27

## 需求结论

为 `zembra-cli add` 增加 `--role` 参数，用于设置笔记创建角色。默认值为 `Human`，并支持常用大小写和缩写输入。

## 已确认规则

| 问题 | 结论 |
| --- | --- |
| CLI 参数 | 新增 `--role` |
| 默认值 | `Human` |
| 支持输入 | `Agent`、`Human`、`agent`、`human`、`a`、`h` |
| 归一化结果 | 写入数据库前统一为 schema 枚举值 `Agent` 或 `Human` |
| 交互体验 | 非法输入返回清晰错误提示 |

## 本阶段范围

| 对象 | 操作 |
| --- | --- |
| CLI add | 增加 `--role` 参数并解析变体 |
| Repository 调用 | 将归一化后的 role 传给 `create_note()` |
| 测试 | 覆盖默认值、支持变体和非法输入 |

## 暂不包含

- 修改既有笔记 role
- 增加全局配置默认 role
- 支持更多 role 枚举
