# Schema 0.2.0 升级需求澄清

日期：2026.04.27

## 需求结论

将 zembra-cli 消费的共享数据库 schema 从 `v0.1.0` 升级到 `v0.2.0`。本阶段只做正向升级，不实现生产环境兼容迁移。

## 已确认规则

| 问题 | 结论 |
| --- | --- |
| 升级范围 | 只升级数据库 schema 版本 |
| 包版本 | `zembra-cli` 包版本保持 `0.1.0` |
| 兼容性 | 当前是封闭开发，不考虑生产环境迁移兼容 |
| tag 目标 | 完成代码适配后为主仓库创建 `v0.2.0` tag |

## 本阶段范围

| 对象 | 操作 |
| --- | --- |
| `vendor/zembra-schema` | 更新 submodule 指针到 `v0.2.0` |
| `notes.role` | 适配新增创建角色字段，默认 `Human`，允许 `Agent` |
| 测试 | 覆盖 schema 初始化、模型校验和 Repository 写入 |
| 文档 | 更新 README 中固定 schema 版本说明 |

## 暂不包含

- 通用 migration runner
- 生产数据迁移脚本
- CLI 包版本升级
- 导入导出功能实现
